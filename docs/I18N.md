# 🌍 Internacionalización (i18n)

INTAQALAB soporta traducciones a 3 idiomas (Español, Inglés y Alemán) gestionados mediante la librería estándar **@ngx-translate**.

## 1. Reglas Core
- **Sin textos estáticos:** Está totalmente prohibido incluir textos duros (_hardcoded_) directamente en los componentes UI o Templates de Angular.
- **Formato:** Los archivos de idiomas son ficheros JSON que residen estáticamente. 
- **Librería:** Usa `@ngx-translate/core` para proveer los tokens.
- **Pipe / Directiva:** Usa preferentemente el pipe puro `{{ 'KEY' | translate }}` en los HTML para beneficiarte del OnPush Change Detection.

## 2. Nomenclatura de Claves (Namespace)

Para garantizar que el gran número de literales del repositorio no colisionen, la nomenclatura de claves sigue **obligatoriamente** esta estructura jerárquica:

`<DOMINIO>.<SECCIÓN_O_FEATURE>.<COMPONENTE>.<PROPIEDAD>`

**Ejemplos correctos:**
- `TRIAL_PLANNING.MUNITIONS.SERIE_PANEL.TITLE`
- `MASTER_DATA.DIALOGS.UPSERT.CREATE_TITLE`
- `SHARED.BUTTONS.CANCEL`

## 3. Dinamismo y Fallbacks

Si necesitas traducir literales desde TypeScript (ej. para un mensaje de error o notificación del Toast), debes inyectar el servicio `TranslateService`.
Sin embargo, prefiere devolver la clave desde el modelo del Store y que la vista resuelva la traducción para evitar acoplamiento de lógica de visualización dentro de NgRx.
