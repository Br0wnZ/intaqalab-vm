## 1 INTRODUCCIÓN ÍNDICE

```
El presente documento tiene como finalidad describir de manera estructurada y comprensible el proceso de
ejecución de un ensayo de fuego. Para ello, se detalla cómo se desarrolla el ensayo, qué perfiles intervienen
en ella y que información necesitan consultar e informar cada uno para garantizar el correcto desarrollo del
ensayo conforme a los procedimientos establecidos. Facilitando una visión global del ensayo desde una
perspectiva tanto operativa como organizativa.
```
### 1.1 Objeto

```
El objetivo del presente documento es definir el proceso completo de ejecución de un ensayo de fuego,
estableciendo con claridad los diferentes perfiles de usuario que participan en él y los datos que cada uno
necesita visualizar y/o consultar. Siendo el enfoque principal garantizar un acceso adecuado a la información
durante el desarrollo del ensayo, proporcionando una base sólida para su correcta supervisión y control.
```
### 1.2 Alcance

```
El alcance del presente documento se centra en el análisis funcional del ensayo de fuego y la identificación
de todos los perfiles que participan. Se especifica qué información necesita cada uno de ellos y en qué
momento del trascurso del ensayo debe consultarla. Quedan excluidos del alcance los cálculos técnicos
derivados del ensayo, los cuales se realizarán fuera del ámbito de este documento, no formando parte de su
contenido.
```

## 3 PERFILES

```
En el desarrollo de un ensayo de fuego se han identificado diferentes perfiles de usuarios que intervienen en
la misma. Cada uno de ellos desempeña funciones específicas y requiere distintos niveles de acceso a la
información generada. Atendiendo al grado de intervención durante el desarrollo del ensayo, los perfiles se
clasifican en dos grandes grupos:
```
- Perfiles con **intervención directa,** estos perfiles participan de manera operativa en la ejecución del
    ensayo asumiendo responsabilidades clave sobre el desarrollo del ensayo. Son los encargados de
    registrar los datos recogidos por los diferentes instrumentos de medida, así como la visualización de
    dichos datos para verificar el correcto desarrollo del ensayo. Los perfiles con intervención directa
    identificados son:
       o Jefe de la línea de tiro
       o Ingeniero de ensayos
       o Jefe de las unidades que intervienen en el ensayo (Balística, Municiones, Armamento,
          Topografía)
       o Técnico unidades que intervienen en el ensayo (Balística, Municiones, Armamento,
          Topografía)
- Perfiles de **consulta u observadores** , estos perfiles no intervienen directamente en la ejecución del
ensayo. Su función consiste en la visualización del desarrollo del ensayo, consultando la información
relevante, sin capacidad de modificar datos ni influir en el proceso operativo. Para estos perfiles es
necesario configurar adecuadamente los datos que pueden visualizar tal y como se indica en el
apartado de Configuración de perfiles**_._** Los perfiles de consulta u observación identificados son:
o Director
o Cliente
o Jefe de la unidad de planificación y análisis
o Consultor de ensayos

### 3.1 Perfiles con intervención directa

Se trata de los perfiles que intervienen de forma directa durante la ejecución del ensayo de fuego, es decir, tienen
en mayor o menor medida una relevancia para la realización del ensayo de fuego ya sea gestionando el desarrollo
del ensayo ya sea informando los datos obtenidos de los diferentes instrumentos de medición. A continuación, se
describe de forma general las funciones que desempeña cada uno de los perfiles implicados durante el desarrollo
del ensayo de fuego. Para cada uno de los perfiles se detallan los datos que necesitan visualizar y/o introducir en
cada caso, además de las acciones que tiene disponibles.

#### 3.1.1 Jefe de línea de tiro

```
Corresponde al perfil encargado de controlar el desarrollo del ensayo de fuego. Para ello debe tener una
visión global de todas las unidades que intervienen en el ensayo para verificar que están operativas y puede
llevarse a cabo el ensayo.
```
#### 3.1.2 Ingeniero de ensayo

```
Este perfil de usuario será el encargado de supervisar la ejecución del ensayo y de garantizar que los
resultados obtenidos cumplen con los criterios de aceptación y rechazo establecidos para el ensayo
planificado. De igual forma, puede realizar ajustes o modificaciones sobre el desarrollo del ensayo o la
planificación de este, ya sea por necesidades técnicas del centro o por petición del cliente. Su labor se
adapta en todo momento a los datos que se van obteniendo durante el ensayo.
```
#### 3.1.3 Unidades intervinientes

```
Por cada una de las unidades que participan activamente en el desarrollo del ensayo de fuego, se identifican
dos tipos de perfiles diferentes en función de las tareas que realizan, por un lado, el perfil de jefe de la

### 3.2 Perfiles de consulta u observadores

```
Se trata de perfiles que no tienen una participación directa en la ejecución operativa del ensayo, pero
necesitan disponer de acceso a la información generada durante su desarrollo. Los datos que pueden
visualizar cada uno de ellos deben configurarse como se detalla en el apartado Configuración de perfiles,
garantizando siempre la trazabilidad y la integridad de los datos.
```
#### 3.2.1 Director

```
Se trata de un perfil de consulta para los directivos del centro de ensayos por lo que deben tener una visión
lo más completa del desarrollo de esta, pero sin posibilidad de realizar modificaciones o cambios en los
datos introducidos, ni intervenir en el desarrollo del ensayo.
```
#### 3.2.2 Jefe área de ensayos de armamento

```
Se trata de un perfil encargado de la supervisión del desarrollo del ensayo de fuego, puede visualizar todos
los datos de esta pero no puede introducir o modificar ninguno de ellos.
El jefe de área de ensayos puede interrumpir el desarrollo del ensayo si observa cualquier causa que impide
el desarrollo normal del ensayo, de igual forma, una vez solventado la causa que motiva la interrupción del
ensayo, podrá informar la finalización de la interrupción permitiendo continuar la ejecución del ensayo.
```
#### 3.2.3 Cliente

```
Se trata del perfil de consulta u observador más importante, es el perfil empleado por el cliente para
supervisar el desarrollo del ensayo. Tendrá una visión limitada de los datos del ensayo para que pueda
tomar las decisiones oportunas sobre el ensayo, pudiendo solicitar al ingeniero de ensayo la modificación
de esta (añadiendo o eliminando medidas, series, disparos...etc.).
```
#### 3.2.4 Jefe de la unidad de planificación y análisis

```
Se trata de un perfil de consulta para la unida de planificación del centro de ensayos por lo que deben tener
una visión lo más completa del desarrollo de esta, pero sin posibilidad de realizar modificaciones o cambios
en los datos introducidos, ni intervenir en el desarrollo del ensayo.
```
#### 3.2.5 Consultor de ensayos

```
Se trata de un perfil de consulta para los consultores de ensayos del centro de ensayos por lo que deben
tener una visión lo más completa del desarrollo de esta, pero sin posibilidad de realizar modificaciones o
cambios en los datos introducidos, ni intervenir en el desarrollo del ensayo.
```

### 4.3 Asignación de equipos

```
Este módulo permite asignar los equipos de medida que se van a utilizar durante la prueba a las series y
disparos que la componen.
Al pulsar el botón "Selector equipos" , se abrirá un modal de asignación cuya interfaz estará organizada
mediante etiquetas (tags) en la parte superior. La disponibilidad y visibilidad de estos tags es dinámica y
depende tanto del rol del usuario como de la planificación previa.
```
#### 4.3.1 Visibilidad de etiquetas por Unidad

```
La visualización de las etiquetas estará restringida según la unidad técnica a la que pertenezca el usuario
logado. Los usuarios con rol de Jefe Unidad Planificación y Análisis y Administrador tendrán acceso a
la totalidad de las etiquetas. Para el resto, la distribución es la siguiente:
```
- **Unidad de Balística (Técnico y Jefe):** Visualizarán los tags **Presión Sensores Piezoeléctricos,**
    **Velocidad Inicial, Trayectografía, Sonido, Vídeo AV** y **Vídeo C**.
- **Unidad de Armamento (Técnico y Jefe):** Visualizarán el tag **Longitud**.
- **Unidad de Municiones (Técnico y Jefe):** Visualizarán los tags **Presión Manómetros, Presión IPG,**
    **Peso** y **Acondicionamiento**.
- **Unidad de Topografía y Jefe de Ensayos de Fuego:** Visualizarán el tag **Tiempo**.

#### 4.3.2 Habilitación de etiquetas y funcionamiento

```
Los tags se habilitarán o deshabilitarán según las medidas o registros solicitados en el módulo de
planificación. Cada medida del catálogo tiene asociados uno o varios equipos necesarios para su
ejecución. Si una medida planificada requiere un equipo perteneciente a una familia específica, el tag
correspondiente se mostrará habilitado en el modal.
Al seleccionar una etiqueta, el usuario visualizará una tabla específica con los equipos disponibles en
Calibry para esa familia.

    o **Balística**

asignado a su perfil.

#### 5.1.2 Estado del disparo

En el extremo derecho aparece un indicador de estado que muestra la situación del disparo actualmente
visualizado. Los estados posibles son:

- Ejecutado
- En curso
- Pendiente

Este estado se actualiza automáticamente conforme avanza la ejecución de la prueba y siempre refleja el
estado real del disparo seleccionado.

#### 5.1.3 Selector de serie y disparo

Se habilitarán selectores para series y disparos. Estos desplegables permiten al usuario navegar entre las
distintas series y los disparos que las componen, facilitando la introducción de datos en el disparo deseado.

Se podrá seleccionar una única serie y disparo, excepto en el caso del acondicionamiento de munición y
datos de armamento, que se podrán seleccionar varias series o disparos.

_Además, el botón “Disparo actual” permite que el sistema lleve al usuario directamente al disparo que está_
en curso, evitando búsquedas manuales.

### 7.3 Disparos EMT

```
El disparo equivalente es una forma de expresar el desgaste real que sufre un tubo teniendo en cuenta que
no todos los disparos generan el mismo nivel de degradación.
Un disparo realizado con:
```
- mayor o menor carga de proyección,
- una temperatura de condicionamiento diferente,
- una presión distinta

```
producirá un desgaste muy diferente en el tubo, aunque físicamente siga contando como “un solo disparo”.
Para poder comparar de manera homogénea el desgaste acumulado en condiciones variables, se utiliza el
concepto de disparo equivalente.
Este convierte cada disparo real en el número de disparos que habrían generado el mismo desgaste si se
hubieran efectuado en condiciones de referencia o condiciones normales definidas por el fabricante o la
institución (carga nominal, temperatura estándar, presión nominal, etc.).
```
#### 7.3.1 Cálculo de disparos equivalente (disparo EMT)

```
El cálculo del disparo equivalente se realiza tras cada disparo ejecutado. Este cálculo permite transformar
un disparo real en el número de disparos que provocarían el mismo desgaste en condiciones estándar.
Para ello, se compara la información del disparo con los valores definidos en el catálogo Gestión del factor
de conversión.
El procedimiento es el siguiente:
```
1. **Identificación del arma o tubo**. En primer lugar, se verifica el arma/tubo con el que se ha
    realizado el disparo y se seleccionan las entradas del catálogo asociadas a dicho arma.
**2. Determinación de la carga empleada.** A continuación, se identifica la **carga de proyección**
    **utilizada** en el disparo. Este es el segundo filtro principal para localizar los posibles factores
    aplicables.
**3. Comprobación de la zona o módulo (si aplica).** Según el modelo de carga:
    a. Si la carga utiliza **zonas** , se compara la zona de carga utilizada durante la ejecución.
    b. Si la carga utiliza **módulos** , se comprueba el número de módulos empleados.
    c. Solo se valida la condición correspondiente (zona o módulo), nunca ambas.


```
Página: 57 de 58 Edición: 0 X
```
4. **Comprobación de la temperatura (si existe).** Si la carga dispone de un valor de **temperatura**
    **de acondicionamiento** , se verifica que el disparo coincide con la temperatura registrada o con
    el rango definido.
**5. Comprobación de la presión (criterio más restrictivo).** Finalmente, se verifica la **presión**
    **media obtenida en el disparo**. Se compara con la **presión de sobrepresión** establecida para
    esa combinación de arma y carga. Este es el criterio más restrictivo: Si la presión media supera
    el umbral de sobrepresión, se debe seleccionar el **factor asociado al caso de mayor**
    **severidad**.
**6. Obtención del factor de conversión** Con todas las condiciones anteriores (arma, carga,
    zona/módulo, temperatura y presión) se identifica el **factor de conversión** correspondiente del
    catálogo. Este factor representa cuántos disparos equivalentes al estándar produce el disparo
    real.
**7. Cálculo del disparo equivalente.** El cálculo es:

```
Disparo EMT = 1 × Factor
```
**8. Actualización del desgaste acumulado.** El resultado del disparo EMT se **suma al total**
    **acumulado** en Calibry:

```
Disparos EMT acumulados = Disparos EMT acumulados + Disparo EMT
```
```
Este proceso se repite después de cada disparo , de modo que Calibry mantiene siempre actualizado
el número total de disparos equivalentes atribuibles al tubo.
```
